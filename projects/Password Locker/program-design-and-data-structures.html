Project: Password Locker

Automating the Boring Stuff with Python



    1. Program Design and Data Structures

      2. We want to be able to run this program with a command line
      argument, such as email or blog.

      • The accounts password will be copied into the clipboard
        so that the user can paste it into a Password field.
      • This way, we can have long, complicated passwords without
        having to memorize them.

      #! python3

      #vipergrip.py - an insecure password locker program.

      PASSWORDS = {
      'email' : 'F7minlBDDuvMJuxESSKHFhTxFtjVB6',
      'blog' : 'VmALvQyKAxiVH5G8v01if1MLZF3sdt',
      'luggage' : '12345'}

    2. Handle Command Line Arugments

      3. Command line arugments will be stored in the variable sys.argv

      • The first item in the sys.argv list should always be the a string containing the programs file name (in this case, 'vipergrip.py).
      • For this program, the arugment is the name of the account whose password we want. Since the command line is mandatory, you will display a message to the user if they forget to add to it.

          • That is, if the sys.argv list has fewer than 2 values in it.

        #! python3
        #vipergrip.py - an insecure password locker program.

        PASSWORDS = {
        'email' : 'F7minlBDDuvMJuxESSKHFhTxFtjVB6',
        'blog' : 'VmALvQyKAxiVH5G8v01if1MLZF3sdt',
        'luggage' : '12345'}


        import sys

        if len(sys.argv) < 2:
        print('Usage: python vipergrip.py [account] - copy account password')
        sys.exit()

        account = sys.argv[1] #first command line argument is the account name

    3. Copy the Right Password

      4. Now that account name is stored, we should be able to see it in the variable account,
      and so we check to see if it exists in the PASSWORDS dictionary as a key.

      • If so, you want to copy the key's value to the clipboard using pyperclip.copy().
      • Note: You don't actually need the account variable; you can just use sys.argv[1]
        everywhere account is used in the program. However, a variable named 'account' is much more
        readable than something cryptic like sys.argv[1]

        #! python3
        #vipergrip.py - an insecure password locker program.

        PASSWORDS = {
        'email' : 'F7minlBDDuvMJuxESSKHFhTxFtjVB6',
        'blog' : 'VmALvQyKAxiVH5G8v01if1MLZF3sdt',
        'luggage' : '12345'}


        import sys
        import pyperclip


        if len(sys.argv) < 2:
        print('Usage: python vipergrip.py [account] - copy account password')
        sys.exit()

        account = sys.argv[1] #first command line argument is the account name

        if account in PASSWORDS:
        pyperclip.copy(PASSWORDS[account])
        print('Password for ' + account + ' copied to clipboard.')
        else:
        print('There is no account name ' + account)
